== README AOM Voting App v1.0 (2015-03-31)

This README describes the basic technical information about the AOM Voting App v1. Ruby version: ruby 2.0.0p353 (2013-11-22 revision 43784) Rails version: Rails 4.0.13 Database: PostgresSQL

This application is built using Ruby on Rails and deployed to production using Heroku and PostgreSQL database. It consists of 6 views accessible to the users and 1 view accessible only for the administrator.

The 6 user views are: /nominees/new /candidate_selection /chairman_selection /fellows_vote /results

The administration view is: /nominees To display the total content of this view you need to click in the 2015 link. This is mainly to prevent unauthorized access to other users. In the future a username/password may be incorporated to make it really secure.

Models

The App contains 3 models: 1. members: A table with 5 columns: id, mail, created_at, updated_at, voted.

  • mail: is the mail address of a member. At the time there are 193 mail addresses registered. To include a new address just go to db/migrate/seeds.rb and include the line Member.create(:mail => 'type here the new email address'). Save and make sure to run in the console rake db:seeds and then rake db:migrate. Finally run the same code in Heroku by typing heroku run rake db:seeds and heroku run rake db:migrate.

  • voted: of boolean type to keep track if the member already voted.

  1. votes: A table with 4 columns: vote_id, vote_type, created_at and updated_at.
  2. vote_id: Type integer. There are 3 votes id (1, 2 and 3). Each id is associated with one type of vote.

  3. vote_type: Type string. The 3 different types are: "Confirmed", "Not Confirmed" and "Abstained".

  4. nominees: A table where each row is a different nominee. All of the app relies on this model. The columns (attributes) are:

  5. mail: Type String. Corresponds to the email of the nominee.
  6. name: Type String. Corresponds to the name of the nominee.
  7. cv: Type Text. Corresponds to the cv of the nominee. For the future this might be wanted to replaced by a pdf.
  8. statement: Type Text. Corresponds to the statement of the nominee. For the future this might be wanted to replaced by a pdf.
  9. committee_check: Type Boolean. This is true once a committee member approves this candidate. Those nominees who have this field as true are displayed in the chairman_selection view.
  10. chairman_check: Type Boolean. This is true once the Chairman approves this candidate. Those nominees who have this field as true are displayed in the fellows_vote view.
  11. confirmed: Type Integer. Keeps the count of the number of times that this nominee has been voted as "Confirmed" in the Fellows_vote view.
  12. not_confirmed: Type Integer. Keeps the count of the number of times that this nominee has been voted as "Not Confirmed" in the Fellows_vote view.
  13. abstained: Type String. Due to an initialization mistake this is of type String instead of Integer. Therefore it requires to be converted to integer where ever it is used (use method .to_i). Keeps the count of the number of times that this nominee has been voted as "Abstained" in the Fellows_vote view.
  14. member_id: Type Integer. Unique identifier for each member. Provided automatically by rails.
  15. nominator_mail: Type String. Records the fellow who nominated this candidate. Although this information is not displayed within any of the user views it might be helpful in case the candidate is selected as Fellow member.
  16. committee_check_count: Type Integer. Keeps the count of the number of times that this nominee has been approved within the candidate_selection view.
  17. vote_id: Type Integer. Records the type of the last vote received by each candidate. It is used to feed the columns confirmed, not_confirmed and abstained.
